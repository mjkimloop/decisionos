# Case Review Checklist

## 목적
이 체크리스트는 케이스 검토 시 누락 없이 모든 중요 항목을 확인하기 위한 표준 절차입니다.

---

## Pre-Review (검토 전 준비)

### 기본 확인
- [ ] 케이스 ID 및 우선순위 확인
- [ ] SLA 남은 시간 확인
- [ ] 본인의 스킬/권한으로 처리 가능한지 확인
- [ ] 이전 노트 및 히스토리 검토
- [ ] 필요한 시스템 접근 권한 확보

---

## 1. 신청 정보 확인

### 기본 정보
- [ ] **신청자 신원**: 이름, 생년월일, 연락처 확인
- [ ] **신청 내용**: 제품, 금액, 기간, 용도
- [ ] **신청 채널**: 온라인/오프라인, 파트너 여부
- [ ] **신청 일시**: 최근 신청인지, 재신청인지 확인

### 완전성 검증
- [ ] 필수 정보 모두 입력되었는가?
- [ ] 정보 간 일관성이 있는가? (예: 생년월일과 나이 일치)
- [ ] 명백한 오류나 의심스러운 정보는 없는가?

---

## 2. 신용 정보 검토

### 신용 점수
- [ ] **신용 점수**: 값 확인 및 기준 비교
  - 점수: _______ / 기준: _______
  - 신용평가기관: NICE / KCB / 기타
  - 조회 일자: _______
- [ ] 신용 점수가 경계선인 경우 (기준 ±10점) 기타 요인 면밀히 검토
- [ ] 신용 점수 변동 추이 확인 (최근 3개월)

### 신용 이력
- [ ] **신용 거래 이력 기간**: _______ 년/월
  - ✅ 충분한 이력 (≥ 12개월)
  - ⚠️ Thin file (< 12개월) → 대체 데이터 검토
  - ❌ 이력 없음 → 거부 검토

- [ ] **연체 이력**:
  - ✅ 연체 없음
  - ⚠️ 경미한 연체 (30일 이하, 1년 이상 경과)
  - ❌ 심각한 연체 (90일 이상, 최근 12개월 내)

- [ ] **현재 연체 여부**:
  - ✅ 현재 연체 없음
  - ❌ 현재 연체 중 → 즉시 거부

- [ ] **신용회복/파산 이력**:
  - ✅ 이력 없음
  - ⚠️ 5년 이상 경과, 이후 정상 거래
  - ❌ 최근 5년 이내 → 거부

### 현재 부채 상황
- [ ] **총 부채액**: _______ 원
- [ ] **대출 건수**: _______ 건
- [ ] **주요 대출 내역**:
  - 금융기관 1: _______ 원
  - 금융기관 2: _______ 원
  - 카드 미결제: _______ 원

---

## 3. 소득 정보 검토

### 소득 수준
- [ ] **연 소득**: _______ 원
- [ ] **소득 유형**:
  - [ ] 근로소득 (정규직/계약직)
  - [ ] 사업소득 (자영업)
  - [ ] 기타 소득 (임대, 투자 등)

- [ ] **최소 소득 기준 충족 여부**:
  - 기준: _______ 원
  - 실제: _______ 원
  - ✅ 충족 / ⚠️ 근접 / ❌ 미달

### 소득 증빙
- [ ] **제출 서류**:
  - [ ] 근로자: 원천징수영수증 / 재직증명서 / 급여명세서
  - [ ] 자영업자: 소득금액증명원 / 부가세신고서 / 사업자등록증
  - [ ] 기타: 해당 소득 증빙

- [ ] **서류 검증**:
  - ✅ 진위 확인 (발급 기관 확인)
  - ✅ 최신 서류 (3개월 이내)
  - ✅ 신청자 본인 명의
  - ❌ 의심스러운 점 있음 → 추가 확인

### 소득 안정성
- [ ] **재직 기간** (근로자):
  - ≥ 3년: 매우 안정적
  - 1-3년: 안정적
  - < 1년: 주의
  - 최근 이직: 재직증명서 확인

- [ ] **사업 기간** (자영업자):
  - ≥ 3년: 안정적
  - 1-3년: 보통
  - < 1년: 위험

- [ ] **소득 추세**:
  - 증가: 긍정적
  - 유지: 안정적
  - 감소: 주의 (사유 확인)

---

## 4. 부채 비율 평가

### DTI (Debt-to-Income Ratio)
- [ ] **계산**:
  - 연간 총 부채 상환액: _______ 원
  - 연 소득: _______ 원
  - DTI = _______ %

- [ ] **평가**:
  - ✅ DTI < 40%: 양호
  - ⚠️ 40% ≤ DTI < 50%: 주의
  - ❌ DTI ≥ 50%: 거부 검토

### DSR (Debt Service Ratio)
- [ ] **규제 한도 확인**:
  - 규제 한도: _______ %
  - 실제 DSR: _______ %
  - ✅ 한도 내 / ❌ 한도 초과

- [ ] **신규 대출 포함 시**:
  - 신규 DSR: _______ %
  - ✅ 한도 내 / ❌ 한도 초과

---

## 5. 담보 및 보증 검토

### 담보 (해당 시)
- [ ] **담보 유형**:
  - [ ] 부동산
  - [ ] 차량
  - [ ] 예금
  - [ ] 기타: _______

- [ ] **담보 가치**:
  - 감정가: _______ 원
  - 대출액: _______ 원
  - LTV: _______ %

- [ ] **담보 검증**:
  - ✅ 소유권 확인
  - ✅ 선순위 담보 없음 또는 확인
  - ✅ 적절한 LTV (일반적으로 ≤ 70-80%)

### 보증 (해당 시)
- [ ] **보증인 정보**:
  - 이름: _______
  - 관계: _______
  - 신용점수: _______

- [ ] **보증인 적격성**:
  - ✅ 신용점수 기준 충족
  - ✅ 소득 충분
  - ✅ 보증 의사 확인

---

## 6. 사기 징후 확인

### 의심 패턴
- [ ] **신원 확인**:
  - ✅ 신분증 진위 확인
  - ✅ 본인 인증 완료
  - ⚠️ 의심스러운 점 없음

- [ ] **서류 진위**:
  - ✅ 공식 발급 기관 확인 가능
  - ⚠️ 위조 의심 없음

- [ ] **행동 패턴**:
  - ⚠️ 단기간 다수 신청 없음
  - ⚠️ 비정상적 접속 (위치, 기기) 없음
  - ⚠️ 급하게 서두르는 행동 없음

- [ ] **정보 일관성**:
  - ✅ 제공 정보 간 일관성 있음
  - ✅ 이전 신청 정보와 일치

### 사기 스코어 (시스템 제공 시)
- [ ] Fraud Score: _______ (0-100)
  - ✅ < 30: 정상
  - ⚠️ 30-70: 주의
  - ❌ > 70: 거부 검토

---

## 7. 정책 및 규제 준수

### 내부 정책
- [ ] **제품 정책 준수**:
  - [ ] 최소/최대 금액 범위 내
  - [ ] 연령 요건 (예: 만 19세 이상)
  - [ ] 기타 제품별 요건

- [ ] **리스크 정책**:
  - [ ] 포트폴리오 집중도 (단일 차주/업종/지역)
  - [ ] 익스포저 한도
  - [ ] 리스크 등급별 한도

### 외부 규제
- [ ] **법적 요구사항**:
  - [ ] 본인 동의 확인
  - [ ] 개인정보 처리 동의
  - [ ] 신용정보 조회 동의

- [ ] **규제 준수**:
  - [ ] DSR 규제 (해당 시)
  - [ ] 총량 규제 (해당 시)
  - [ ] 기타 규제 (업종별)

### AML/CFT
- [ ] **자금세탁방지**:
  - ✅ AML 스크리닝 통과
  - ✅ 제재 리스트 매칭 없음
  - ✅ PEP (정치적 주요 인물) 해당 없음

- [ ] **자금 출처**:
  - ✅ 정당한 자금 출처
  - ⚠️ 의심스러운 점 없음

---

## 8. 거래 이력 검토 (기존 고객)

### 기존 관계
- [ ] **거래 기간**: _______ 년/월
- [ ] **거래 제품**:
  - 제품 1: _______
  - 제품 2: _______

- [ ] **거래 성과**:
  - ✅ 성실 상환 (연체 0회)
  - ⚠️ 경미한 연체 (1-2회, 30일 이하)
  - ❌ 연체 이력 (90일 이상)

- [ ] **수익성**:
  - 총 수익: _______ 원
  - 손실: _______ 원
  - ✅ 수익성 고객 / ⚠️ 보통 / ❌ 손실 고객

---

## 9. 유사 케이스 참조

### 일관성 확인
- [ ] **유사 케이스 검색**:
  - 유사 신용점수/소득/부채 비율 케이스

- [ ] **결정 일관성**:
  - 유사 케이스 승인율: _______ %
  - 본 케이스와 주요 차이점: _______

- [ ] **정책 예외 선례**:
  - 유사한 예외 승인 사례 있는가?
  - 승인 근거: _______

---

## 10. 최종 판단

### 종합 평가
- [ ] **긍정 요인** (체크):
  - [ ] 신용점수 우수 또는 기준 충족
  - [ ] 소득 안정적
  - [ ] 부채 비율 양호
  - [ ] 연체 이력 없음
  - [ ] 담보/보증 충분 (해당 시)
  - [ ] 기존 거래 우수 (해당 시)
  - [ ] 사기 징후 없음

- [ ] **부정 요인** (체크):
  - [ ] 신용점수 부족
  - [ ] 소득 불안정 또는 부족
  - [ ] 부채 비율 높음
  - [ ] 연체 이력 있음
  - [ ] 담보/보증 불충분 (해당 시)
  - [ ] 사기 징후 있음
  - [ ] 정책/규제 위반

### 결정
- [ ] **승인 (Approve)**
  - 조건: _______
  - 한도: _______ 원
  - 금리: _______ %
  - 기간: _______ 개월
  - 특이사항: _______

- [ ] **거부 (Deny)**
  - Reason Codes: _______
  - 주요 사유: _______
  - 개선 방안: _______

- [ ] **서류 요청 (Request Documents)**
  - 필요 서류: _______
  - 제출 기한: _______

- [ ] **에스컬레이션 (Escalate)**
  - 사유: _______
  - 권장 결정: _______

### 문서화
- [ ] **Case Note 작성**:
  - 검토 내용 요약
  - 주요 고려 사항
  - 결정 근거
  - 추가 조치 사항

- [ ] **Reason Codes 입력**:
  - 승인 시: 승인 근거 코드
  - 거부 시: 거부 사유 코드 (우선순위순)

- [ ] **첨부 파일 확인**:
  - 필요한 증빙 서류 모두 첨부
  - 파일 스캔 완료 (AV/PII)

---

## Post-Review (검토 후 확인)

### 최종 체크
- [ ] 모든 필수 항목 체크 완료
- [ ] 결정이 정책 및 규제 준수함
- [ ] 문서화 완료 (Note, Reason Codes)
- [ ] 고객 통지 준비 완료
- [ ] SLA 내 처리 완료

### 제출
- [ ] 결정 제출
- [ ] 케이스 상태 업데이트
- [ ] 다음 작업 할당 (해당 시)

---

## 특수 상황 체크리스트

### 경계선 케이스 (Borderline)
- [ ] 기준에서 ±5% 범위의 케이스
- [ ] 긍정/부정 요인 균형 평가
- [ ] 유사 케이스 일관성 중요
- [ ] Senior Reviewer 에스컬레이션 고려

### 정책 예외 요청
- [ ] 예외 승인 권한 확인
- [ ] 예외 근거 명확히 문서화
- [ ] 리스크 완화 방안 확인
- [ ] Senior Reviewer 또는 Ops Admin 승인 필요

### VIP 고객
- [ ] VIP 상태 확인
- [ ] 우대 조건 확인
- [ ] 신속 처리 (P0/P1)
- [ ] 고급 서비스 제공

### 고액 거래
- [ ] 권한 한도 확인
- [ ] 추가 검증 절차 (필요시)
- [ ] 리스크 집중도 확인
- [ ] Senior Reviewer 승인 (필요시)
