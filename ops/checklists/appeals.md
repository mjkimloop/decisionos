# Appeals Review Checklist

## 목적
이 체크리스트는 이의 제기(Appeals) 케이스를 검토할 때 공정하고 철저한 재평가를 보장하기 위한 표준 절차입니다.

---

## Appeals 기본 원칙

### 핵심 원칙
1. **독립성**: 원래 검토자와 다른 Senior Reviewer가 검토
2. **공정성**: 모든 증빙을 새롭게 검토, 선입견 배제
3. **투명성**: 결정 근거를 명확히 문서화
4. **신속성**: 고객 불편 최소화를 위해 신속 처리 (p95 ≤ 72시간)

---

## Pre-Review (검토 전 준비)

### Appeals 적격성 확인
- [ ] **제출 기한 확인**:
  - 원래 결정 일자: _______
  - 이의 제기 일자: _______
  - 기한 내 제출 (일반적으로 30일 이내): ✅ / ❌

- [ ] **제출 자격 확인**:
  - 신청자 본인 또는 법적 대리인
  - 신원 확인 완료

- [ ] **이의 제기 사유 확인**:
  - [ ] 새로운 증빙 제출
  - [ ] 오류 주장 (계산, 정보, 정책 적용)
  - [ ] 특별한 사정 설명
  - [ ] 기타: _______

- [ ] **중복 제출 여부**:
  - ✅ 최초 제출
  - ⚠️ 재제출 (이전 Appeals 결과 확인)

### 이해 상충 확인
- [ ] **원래 검토자 확인**:
  - 원래 검토자: _______
  - 본인과 동일인물: ✅ / ❌
  - ❌인 경우에만 진행, ✅인 경우 재배정

- [ ] **개인적 이해관계**:
  - ✅ 이해관계 없음
  - ❌ 이해관계 있음 (친인척, 업무 관계 등) → 재배정

## 4. 고객 주장 검토

### 오류 주장 확인
- [ ] **고객 주장**: _______

- [ ] **오류 유형 (해당 시)**:
  - [ ] 계산 오류 (DTI, 소득, 부채 등)

  - [ ] 정책 적용 오류 (기준 또는 예외 조항)

- [ ] **오류 영향 평가**:
  - [ ] 결정에 중대한 영향 (번복 사유)
  - [ ] 일부 영향 (조건 조정 가능)

### 특별한 사정 검토

- [ ] **특별 사정 유형**:

  - [ ] 외부 요인 (경제 위기, 팬데믹 등)
  - [ ] 기술적 문제 (서류 제출 오류 등)

- [ ] **사정의 타당성**:
  - ✅ 타당하고 증빙 있음

## 6. Appeals 결정

### 결정 유형

#### Option 1: Uphold (유지) ✅
**조건**: 원래 결정이 정당하고, 새로운 증빙으로도 기준 충족 불가

- [ ] **유지 사유**:
  - [ ] 새로운 증빙 불충분
  - [ ] 여전히 기준 미달
  - [ ] 오류 주장 타당하지 않음
  - [ ] 특별 사정 인정 어려움
  - [ ] 정책/규제상 불가

- [ ] **문서화**:
  - Reason Codes (원래 코드 유지 또는 업데이트)
  - 유지 근거 상세 설명
  - 고객에게 명확한 설명 제공
  - 개선 방안 제시 (재신청 조건)

- [ ] **고객 통지**:
  - 유지 결정 통지
  - 상세 설명서 발송
  - 추가 질문 대응 준비

#### Option 2: Overturn (번복) ✅
**조건**: 새로운 증빙 또는 오류 수정으로 승인 기준 충족

- [ ] **번복 사유**:
  - [ ] 새로운 증빙으로 기준 충족
  - [ ] 원래 결정에 오류 발견
  - [ ] 정책 변경으로 승인 가능
  - [ ] 특별 사정 인정 (정책 예외)

- [ ] **승인 조건 설정**:
  - 한도: _______ 원
  - 금리: _______ %
  - 기간: _______ 개월
  - 특별 조건: _______
    - [ ] 담보 필요
    - [ ] 보증인 필요
    - [ ] 소득 증빙 정기 제출
    - [ ] 기타: _______

- [ ] **문서화**:
  - 번복 근거 명확히 기록
  - 새로운 Reason Codes (승인 근거)
  - 원래 결정과의 차이점 설명
  - 교훈 및 프로세스 개선 사항

- [ ] **고객 통지**:
  - 승인 통지 및 조건 안내
  - 새로운 설명서 발송
  - 계약 프로세스 안내

#### Option 3: Partial (조건부) ✅
**조건**: 완전 승인은 어려우나 조건부로 승인 가능

- [ ] **조건부 사유**:
  - [ ] 담보/보증 추가 시 승인 가능
  - [ ] 한도 축소 시 승인 가능
  - [ ] 추가 서류 제출 시 승인 가능
  - [ ] 특정 조건 충족 시 승인 가능

- [ ] **조건 설정**:
  - 필수 조건: _______
  - 한도: _______ 원 (축소 또는 제한)
  - 금리: _______ %
  - 조건 충족 기한: _______

- [ ] **문서화**:
  - 조건부 결정 근거
  - 조건 상세 설명
  - 조건 충족 시 진행 절차

- [ ] **고객 통지**:
  - 조건부 결정 통지
  - 조건 명확히 설명
  - 조건 충족 방법 안내
  - 질문 대응

## Post-Review (검토 후 조치)

### 결정 제출
- [ ] Appeals 결정 제출
- [ ] 케이스 상태 업데이트
- [ ] 고객 통지 자동 발송 확인

### 후속 조치
- [ ] **Overturn 시**:
  - [ ] 계약 프로세스 시작
  - [ ] 관련 팀에 통지
  - [ ] 조건 충족 추적 (조건부 시)

- [ ] **Uphold 시**:
  - [ ] 추가 문의 대응 준비
  - [ ] 재신청 조건 명확히 안내

### 학습 및 개선
- [ ] **케이스 리뷰**:
  - [ ] 유사 케이스 증가 추세 모니터링
  - [ ] 정책 조정 필요성 검토
  - [ ] 자동화 개선 기회 식별

- [ ] **지식 공유**:
  - [ ] 복잡한 케이스는 팀 공유
  - [ ] 베스트 프랙티스 문서화
